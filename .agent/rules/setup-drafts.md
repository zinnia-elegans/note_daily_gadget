# Setup Drafts from Schedule
## 概要
[03_schedule/schedule_2026.md](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/03_schedule/schedule_2026.md:0:0-0:0) から指定された期間（例:「1月」）の記事タイトルを読み込み、記事生成に必要な「ドラフトファイル（空箱）」を一括で作成するワークフロー。
## 手順
1.  **対象の特定**
    - ユーザーから指定された期間のタイトルと掲載日をスケジュールから抽出する。
2.  **ドラフト生成ループ（各記事に対して実行）**
    1.  **ファイル作成**: `02_article/YYYY-MM-DD_タイトル.md` を作成する。
    2.  **メタデータ更新**: [02_article/_metadata.json](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/02_article/_metadata.json:0:0-0:0) にエントリを追加する。
            - **重要**: `url` フィールドには**空文字 `""`** を設定すること。架空のURLを生成してはならない。
            - `published_date` はスケジュールの日付を使用する。
            - `category` と `tags` は記事内容から推測して埋める。
    3.  **商品仮選定 (AI Action)**:
         - タイトルに基づき、Amazon等で人気がありそうな具体的な商品名を **6つ** ピックアップする。
         - これらはあくまで「仮」であり、ユーザーが後で書き換える前提で作成する。
    4.  **テンプレート適用**:
         - 以下のMarkdown構成で内容を記述する。ユーザーへの指示コメントを含む。
    ```markdown
    # {タイトル}
    <!-- ここに導入文が入ります（AI生成） -->
    ## ① {仮の商品名1}
    <!-- ↓ここにURLと要約を貼り付けてください -->
    ## ② {仮の商品名2}
    <!-- ↓ここにURLと要約を貼り付けてください -->
    ...（6つ分繰り返し）...
    ## あわせて読みたい
    ## 最後に
    ```
3.  **完了報告**
    - 生成したファイル名の一覧を表示する。
